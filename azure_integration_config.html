TestDriver MCP supports multiple ways to integrate with Azure DevOps. Each method has different security levels, performance characteristics, and use cases. Select the method that best fits your organization's requirements.
üîë Personal Access Token (PAT)
Most common and straightforward method. Uses a token generated in Azure DevOps for authentication.
‚úì Advantages
  • Easy to set up

‚úó Disadvantages

‚öôÔ∏è Step 2: Configure Your Integration
Personal Access Token (PAT) Configuration
Generate a PAT token in Azure DevOps and paste it below. Keep this token secure and never share it.
Example: https://dev.azure.com/contoso
Generate at: https://dev.azure.com/yourorganization/_usersSettings/tokens
Scopes needed: Code (Read), Work Items (Read), Build (Read), Release (Read)
Token Expiration: Set your token to expire in 1 year. TestDriver will notify you 30 days before expiration.
Recommended: Encrypts the token using DPAPI (Windows) or system keyring (Linux/Mac)
Managed Identity (Azure Native) Configuration
Use Azure's native identity system. This is only available when running on Azure resources.
Prerequisites:
  • TestDriver running on Azure VM, AKS, or App Service
  • Managed Identity enabled on the Azure resource
  • Appropriate RBAC permissions assigned
Example: https://dev.azure.com/contoso
Only needed for user-assigned identities
‚ö†Ô∏è Setup Required: You must assign RBAC permissions to your managed identity in the Azure DevOps organization. Run these Azure CLI commands:

az role assignment create --assignee {identity-object-id} --role "Contributor" --scope /subscriptions/{subscription-id}
SSH Public Key Configuration
Use SSH key pairs for secure authentication. Works best for Git operations.
Generate with: ssh-keygen -t rsa -b 4096
Keep this secure. Will be encrypted before storage.
Add Public Key to Azure DevOps:
  1. Go to User Settings ‚Üí SSH public keys
  2. Click "Add"
  3. Paste your public key
  4. Click "Save"
Highly recommended for production use
Service Principal (App Registration) Configuration
Enterprise-grade authentication using Azure AD Service Principal.
Find at: Azure AD ‚Üí Properties ‚Üí Directory ID
Create at: App Registration ‚Üí Certificates & secrets ‚Üí New client secret

User-initiated authentication using modern OAuth 2.0 flow.
Must match the URI configured in your app registration
Configure App Registration:

  2. Set Redirect URI to your TestDriver URL
